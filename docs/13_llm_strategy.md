# LLM Usage Strategy

## 1. Budget Overview

- **月度预算上限**: $100
- **目标运行成本**: $20-50/月
- **可接受最低**: $10/月（降低处理量）

---

## 2. Model Selection

### 2.1 推荐模型组合

#### 主力模型：GPT-4o
- **成本**: $2.50/1M输入, $10/1M输出
- **用途**: 
  - 日常摘要生成
  - 相关性判断
  - 主题提取
- **优势**: 性价比最高，质量优秀

#### 备选主力：Claude 3.5 Sonnet
- **成本**: $3/1M输入, $15/1M输出
- **用途**: 同GPT-4o
- **优势**: 长文本理解更好，适合论文分析

#### 高质量模型：GPT-4 Turbo 或 Claude Opus
- **成本**: $10-30/1M输入
- **用途**:
  - 候选方向生成（每周）
  - 复杂推理任务
  - 关键决策分析
- **使用频率**: 每周1-2次

#### 快速筛选：GPT-4o-mini（可选）
- **成本**: $0.15/1M输入, $0.60/1M输出
- **用途**:
  - 初步内容过滤
  - 关键词扩展
  - 简单分类
- **优势**: 成本极低，适合大批量处理


---

## 3. Cost Optimization Strategies

### 3.1 批量处理
- 一次prompt处理多篇相关内容
- 减少API调用固定开销
- 预计节省30-40%成本

### 3.2 分层处理
```
第一层：关键词快速过滤（无成本）
    ↓
第二层：GPT-4o-mini初筛（低成本，可选）
    ↓
第三层：GPT-4o深度分析（主力）
    ↓
第四层：高质量模型关键决策（每周）
```

### 3.3 缓存策略
- 保存所有LLM输出
- 相同内容不重复处理
- 支持重新解析时复用

### 3.4 动态调整
- 根据月度预算动态调整处理量
- 优先处理高权重内容
- 低分内容可跳过LLM处理

---

## 4. Token Usage Estimation

### 4.1 日常处理（假设每天处理25篇）

#### 摘要生成
- 输入：每篇1000 tokens × 25 = 25k tokens
- 输出：每篇300 tokens × 25 = 7.5k tokens
- 成本（GPT-4o）：$0.063 + $0.075 = $0.138

#### 相关性判断
- 输入：每篇500 tokens × 25 = 12.5k tokens
- 输出：每篇100 tokens × 25 = 2.5k tokens
- 成本（GPT-4o）：$0.031 + $0.025 = $0.056

#### 主题提取（批量）
- 输入：10k tokens/天
- 输出：2k tokens/天
- 成本（GPT-4o）：$0.025 + $0.020 = $0.045

**日均成本**: ~$0.24


### 4.2 周度任务

#### 候选方向生成
- 输入：汇总本周高分内容，约30k tokens
- 输出：3个候选方向，约5k tokens
- 成本（GPT-4 Turbo）：$0.30 + $0.30 = $0.60

#### 周报生成
- 输入：统计和趋势数据，约15k tokens
- 输出：结构化报告，约3k tokens
- 成本（GPT-4o）：$0.038 + $0.030 = $0.068

**周均成本**: ~$0.67

### 4.3 月度总计

- 日常处理：$0.24 × 30 = $7.2
- 周度任务：$0.67 × 4 = $2.68
- **月度总计**: ~$10

**结论**：在预算$100的情况下，有充足的余量用于：
- 增加处理量（可处理50-100篇/天）
- 使用更好的模型
- 进行更深度的分析
